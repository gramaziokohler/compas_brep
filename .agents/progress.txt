## Task: Brep thin wrapper — issue 03

Removed Python-owned topology as canonical state from `Brep`. Changes in `brep.py`:

- `__from_data__`: after `brep_rebuild` creates the native OCC shape from v3 Python
  topology, immediately clear `_vertices`, `_edges`, `_loops`, `_faces` and reset
  `_topology_loaded = False`. Native is now the sole source of truth; topology is
  lazily repopulated from it on first access.

- `_replace_from`: added `self._topology_loaded = other._topology_loaded` so that
  in-place operations (flip, transform, etc.) correctly reset the topology-loaded flag
  from the replacement Brep (always False for fresh operation results). Previously,
  `_topology_loaded` was left at True from before the operation, causing subsequent
  topology access to return a stale empty list instead of repopulating from the new native.

- Removed dead `_ensure_native` method (called nowhere; embedded the old rebuild-from-
  Python-topology pattern which is now obsolete).

- Updated module and class docstrings to reflect the thin-wrapper architecture.

The `_deserialize_brep_data` helper function remains: it builds a temporary Python
topology Brep used exclusively by `brep_rebuild` to reconstruct the native OCC shape.
It will be removed in issue 04 when the serialization format changes.

All @pytest.mark.occ tests verified via CI.

All acceptance criteria for issue 03 checked off.

## Task: Rhino backend operational testing — issue 08

Ran comprehensive live testing of every supported Brep operation in the Rhino backend via
the LAMCP bridge (Rhino 8 / Grasshopper). Fixed five bugs found during testing.

### Bugs fixed

**1. `rhino_aabb` — wrong COMPAS Box constructor API**
`Box(Frame(...), xsize, ysize, zsize)` → `Box(xsize, ysize, zsize, frame=Frame(...))`.
The COMPAS API changed; the old positional order raised `TypeError: 'float' object is not
subscriptable`.

**2. `rhino_volume` — negative volume for extruded shapes**
`VolumeMassProperties.Compute.Volume` returns negative for inside-out breps.  Changed to
`abs(mp.Volume)` so volume is always positive (consistent with OCC backend).

**3. `rhino_tessellate` — `NotImplementedError`**
Implemented `rhino_tessellate` using `Rhino.Geometry.Mesh.CreateFromBrep` with
`MeshingParameters`.  Returns `(Mesh, list[Polyline])` matching the OCC convention.
`to_viewmesh()` now works in the Rhino backend.

**4. `rhino_to_step` / `rhino_from_step` — wrong API (`File3dm.WriteToStep` does not exist)**
Replaced with `doc.WriteFile(path, FileWriteOptions(SuppressDialogBoxes=True, ...))` /
`doc.ReadFile(path, FileReadOptions())`.  Note: these must be called from the Rhino/GH main
thread — calling them from the LAMCP bridge HTTP thread blocks Rhino; STEP I/O was not
included in bridge-based tests for this reason.

**5. `brep_to_rhino` reconstruction — three separate bugs causing broken deserialization**

*5a. Face breps not joined correctly:*
The loop was appending individual face-copies to a single empty Brep then joining it against
itself (joining only that one brep, so edges across faces were never sewn).  Changed to
collect each face as a separate Brep and join them all in one `JoinBreps` call.

*5b. `_extract_knots_mults` — too many knot copies added at clamped ends:*
The function added `degree` copies at each end when restoring Rhino's stripped knot
convention, but should only add 1.  Rhino stores knots with the first and last element
stripped once (not `degree` times).  Fixed `full_kv = [first] + raw_knots + [last]`.
This caused `IndexOutOfRangeException` when setting NURBS curve knots for circular arcs.

*5c. `JoinBreps` tolerance too tight for cross-backend NURBS reconstruction:*
`TOL.absolute` in COMPAS is `1e-9`; this caused near-coincident edges (e.g. a
Rhino-reconstructed OCC cylindrical surface) to remain unjoined.  Changed to `1e-6`
(Rhino standard fitting tolerance).

**6. `_trim_nurbs_surface_from_2d` (new helper):**
NURBS surfaces serialized from the Rhino backend store the full (untrimmed) surface plus 2D
parametric trim curves.  On deserialization the surface domain needs to be restricted to the
face boundary before calling `ToBrep()`.  The helper extracts the bounding box of all 2D
trim curve endpoints and calls `NurbsSurface.Trim(u_interval, v_interval)`.
