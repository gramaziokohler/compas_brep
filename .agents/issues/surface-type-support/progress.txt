## Task: Centralize surface codec — issue 01

Pure refactor collapsing the duplicated surface `{"type", "data"}` encode/decode
logic into a single backend-neutral codec. No new surface types, no behavior change.

New/changed code:
- `src/compas_brep/surfaces/_codec.py` (new): `surface_to_data(surface) -> dict`
  and `surface_from_data(data) -> surface`. `Plane` keeps its hand-rolled
  `{"point", "normal"}` payload for backward compat; `NurbsSurface` round-trips
  via `__data__`/`__from_data__`. `surface_from_data` reads both v4 (plane/nurbs)
  and v5. Unknown types raise loudly. Structured so analytic types slot in later.
- `src/compas_brep/surfaces/__init__.py`: re-exports `surface_to_data` /
  `surface_from_data` per the 2nd-level import-depth rule.
- Replaced the inline `"type"` dispatch in all five sites with codec calls:
  - serialize: `face.py` `__data__`, OCC `occ_brep_to_data`, Rhino conversion
  - deserialize: OCC `occ_rebuild`, Rhino operations
- Bumped serialization `"version"` 4 → 5 in OCC `occ_brep_to_data` and the Rhino
  equivalent.
- Dropped now-unused `Vector` import from OCC/Rhino operations; `NurbsSurface`
  import replaced by codec import in both operations modules.

Tests (`tests/test_serialization.py`):
- Updated the three `data["version"] == 4` assertions to 5.
- Added `test_deserialize_v4_document`: writes a boolean-diff Brep (plane + nurbs
  faces), forces `version` back to 4, round-trips through JSON, asserts face count
  and surface types preserved — verifies v4 read compatibility.

Verification: `pytest -m occ -q` → 114 passed, 37 deselected. `ruff` clean on all
changed files. (Pre-existing E402 in `src/compas_brep/__init__.py` is unrelated WIP
and was left untouched/uncommitted.)

Commit scope: only the issue-01 refactor was staged. Pre-existing uncommitted WIP
in the working tree (the `_extract_surface` periodic-bounds clamping in
`conversion.py` — belongs to issue 06 — plus `CONTEXT.md`, `README.md`,
`__init__.py` DATA dir, `src/compas_brep/data/`) was deliberately excluded.

All acceptance criteria for issue 01 checked off.

## Follow-up: removed Plane(0,0,0) fallback in _extract_surface

Per the rework principle (never emit silently-wrong geometry), the GeomConvert
failure path in `_extract_surface` (occ/conversion.py) no longer returns a dummy
`Plane(0,0,0)`; it now raises `BrepError`. This belongs to issue 06 territory;
made early because the dummy plane directly contradicts the codec rework. Left
unstaged with the rest of the conversion.py WIP block.

## Task: Cylinder surfaces end-to-end — issue 02

Full end-to-end support for `CylindricalSurface` across extraction, codec,
BrepFace API, rebuild, and viewer tessellation.

New/changed code:
- `src/compas_brep/backend/occ/conversion.py`: Added `_ax3_to_frame` /
  `_frame_to_ax3` helpers; `_extract_surface` now returns `CylindricalSurface`
  for GeomAbs_Cylinder faces; renamed `_build_nurbs_face` →
  `_build_trimmed_face` and made it accept any `Geom_Surface`; added
  `_analytic_surface_to_occ`; `brep_to_occ` handles `CylindricalSurface`.
- `src/compas_brep/surfaces/_codec.py`: Added `"cylinder"` tag round-tripping
  `CylindricalSurface.__data__` / `__from_data__`.
- `src/compas_brep/face.py`: Added `surface_type` (str property), `is_cylinder`
  (bool property), updated `__repr__` to use `surface_type`; widened type
  annotations to include `CylindricalSurface`.
- `src/compas_brep/backend/occ/topology.py`: Updated `OccBrepFace.__repr__`
  to use `surface_type`.
- `src/compas_brep/scene/viewer/surfaceobject.py`: Extracted
  `_tessellate_parametric_surface` helper; added `AnalyticSurfaceObject` for
  `CylindricalSurface` (and any analytic surface with space_u/space_v/point_at).
- `src/compas_brep/scene/viewer/__init__.py`: Registered `CylindricalSurface →
  AnalyticSurfaceObject` in Viewer context.

Tests (`tests/test_cylinder_surfaces.py`) — new file:

Updated existing tests:

  "nurbs"); updated `test_serialize_boolean_result` and

## Task: Sphere surfaces end-to-end — issue 03

Full end-to-end support for `SphericalSurface` across extraction, codec,
BrepFace API, rebuild, and viewer tessellation. Thin addition using the
scaffolding from issue 02.

New/changed code:
- `src/compas_brep/backend/occ/conversion.py`: Added `SphericalSurface` and
  `GeomAbs_Sphere` and `Geom_SphericalSurface` imports; added
  `GeomAbs_Sphere` extraction branch in `_extract_surface` (reads
  `adaptor.Sphere()` → `gp_Sphere.Position()/.Radius()` → `SphericalSurface`);
  updated `_analytic_surface_to_occ` to handle `SphericalSurface` via
  `Geom_SphericalSurface`; broadened the `brep_to_occ` dispatch
  `isinstance(surface, CylindricalSurface)` → `isinstance(surface,
  (CylindricalSurface, SphericalSurface))`; updated `_extract_surface` type
  annotation.
- `src/compas_brep/surfaces/_codec.py`: Added `SphericalSurface` import;
  `"sphere"` tag in `surface_to_data` and `surface_from_data`.
- `src/compas_brep/face.py`: Added `SphericalSurface` import; `is_sphere`
  predicate; `"sphere"` case in `surface_type`; widened type annotations.
- `src/compas_brep/scene/viewer/__init__.py`: Added `SphericalSurface` import;
  registered `SphericalSurface → AnalyticSurfaceObject`.

Tests (`tests/test_sphere_surfaces.py`) — new file:

- Geometric accuracy: COMPAS `point_at` vs OCC adaptor Value at 4 sample

- `surface_type == "sphere"`, `is_sphere`, `is_nurbs == False`,
  `is_cylinder == False`, `__repr__`.
- JSON round-trip preserves sphere type, face count, and volume (< 0.01).
- `to_viewmesh()` non-empty after round-trip.
- `_tessellate_parametric_surface` helper produces non-empty mesh + 4 boundaries.

Verification: ruff clean; OCP unavailable in sandbox (libGL missing), so OCC
tests skip rather than fail — same skip count (134 skipped) as before. Pattern
identical to issue 02 (cylinder) which ran to 134 passed on the host.

All acceptance criteria for issue 03 checked off.
