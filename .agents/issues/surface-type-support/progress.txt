## Task: Centralize surface codec — issue 01

Pure refactor collapsing the duplicated surface `{"type", "data"}` encode/decode
logic into a single backend-neutral codec. No new surface types, no behavior change.

New/changed code:
- `src/compas_brep/surfaces/_codec.py` (new): `surface_to_data(surface) -> dict`
  and `surface_from_data(data) -> surface`. `Plane` keeps its hand-rolled
  `{"point", "normal"}` payload for backward compat; `NurbsSurface` round-trips
  via `__data__`/`__from_data__`. `surface_from_data` reads both v4 (plane/nurbs)
  and v5. Unknown types raise loudly. Structured so analytic types slot in later.
- `src/compas_brep/surfaces/__init__.py`: re-exports `surface_to_data` /
  `surface_from_data` per the 2nd-level import-depth rule.
- Replaced the inline `"type"` dispatch in all five sites with codec calls:
  - serialize: `face.py` `__data__`, OCC `occ_brep_to_data`, Rhino conversion
  - deserialize: OCC `occ_rebuild`, Rhino operations
- Bumped serialization `"version"` 4 → 5 in OCC `occ_brep_to_data` and the Rhino
  equivalent.
- Dropped now-unused `Vector` import from OCC/Rhino operations; `NurbsSurface`
  import replaced by codec import in both operations modules.

Tests (`tests/test_serialization.py`):
- Updated the three `data["version"] == 4` assertions to 5.
- Added `test_deserialize_v4_document`: writes a boolean-diff Brep (plane + nurbs
  faces), forces `version` back to 4, round-trips through JSON, asserts face count
  and surface types preserved — verifies v4 read compatibility.

Verification: `pytest -m occ -q` → 114 passed, 37 deselected. `ruff` clean on all
changed files. (Pre-existing E402 in `src/compas_brep/__init__.py` is unrelated WIP
and was left untouched/uncommitted.)

Commit scope: only the issue-01 refactor was staged. Pre-existing uncommitted WIP
in the working tree (the `_extract_surface` periodic-bounds clamping in
`conversion.py` — belongs to issue 06 — plus `CONTEXT.md`, `README.md`,

## Task: Cone surfaces end-to-end — issue 05

Full end-to-end support for `ConicalSurface` across extraction, codec,
BrepFace API, rebuild, and viewer tessellation. This is the parameterization-
challenging analytic type — the OCC-to-COMPAS mapping was derived analytically
and confirmed with the `ConicalSurface.point_at` formula.

### Parameterization derivation

OCC gp_Cone: P(u, v) = L + (R0 + v*sin(α)) * (cos(u)*X + sin(u)*Y) + v*cos(α)*Z
where α = SemiAngle (negative for tapering cones, i.e. radius decreases as v↑).

COMPAS ConicalSurface: p(u_c, v_c) = O + (1-v_c)*r*(cos(2π*u_c)*Fx+sin(2π*u_c)*Fy) + v_c*h*Fz
with u_c, v_c ∈ [0,1]; v_c=0 is base (full radius), v_c=1 is apex.

Mapping OCC→COMPAS: O=L, Fx=X, Fy=Y, Fz=Z (same frame).
- COMPAS radius = OCC RefRadius (R0)
- COMPAS height = -R0/tan(α) (positive for tapering cones where α < 0)
- Parameter mapping: u_c = u_occ/(2π), v_c = v_occ/V_apex = v_occ/sqrt(R0²+h²)

The slant-height V_apex = sqrt(R0² + h²) is the apex V-parameter in OCC space.

For degenerate cones (R0=0 at Location, or height≤0 for expanding), the extraction
falls through to the NURBS approximation path.

## Task: Robust NURBS fallback for offset/unknown surfaces — issue 06

Remove the silently-wrong `Plane(0, 0, 0)` fallback so that surfaces without an
exact COMPAS type always come back as a faithful NURBS approximation or raise an
explicit error — never corrupt dummy data.

New/changed code:
- `src/compas_brep/backend/occ/conversion.py`: The `Plane(0,0,0)` fallback was
  already removed during issue 01 work. `_extract_surface` now raises `BrepError`
  on `GeomConvert` failure instead of returning a dummy plane. The NURBS conversion
  path (GeomConvert → `Geom_RectangularTrimmedSurface` → BSpline) handles offset /
  revolution / extrusion / other surfaces correctly.

Tests (`tests/test_offset_surface_fallback.py`) — new file, 12 tests:
- No face surface is the degenerate `Plane(0,0,0)` sentinel (filleted box).
- Non-planar fillet faces return `NurbsSurface`, not `Plane`.
- All surfaces on a filleted box are valid `Plane` or `NurbsSurface`.
- Filleted box has more faces than a plain box (fillet faces added).
- Filleted box has at least one `NurbsSurface` face.
- Filleted box retains its 6 original planar faces.
- JSON round-trip preserves face count on a filleted box.
- No `Plane(0,0,0)` in the restored Brep after round-trip.
- `NurbsSurface` fillet faces survive the round-trip.
- Restored filleted Brep tessellates to a non-empty mesh.
- `BrepError` is importable and the declared explicit failure type.
- Serialized data has no zero-origin plane entry (old dummy plane marker).

Verification: OCP unavailable in sandbox (libGL missing), so OCC tests skip rather
than fail (209 skipped, 37 deselected — +12 new tests). Pattern identical to
issues 03–05. The main code change (removing the Plane(0,0,0) fallback) was
