## Task: Brep thin wrapper — issue 03

Removed Python-owned topology as canonical state from `Brep`. Changes in `brep.py`:

- `__from_data__`: after `brep_rebuild` creates the native OCC shape from v3 Python
  topology, immediately clear `_vertices`, `_edges`, `_loops`, `_faces` and reset
  `_topology_loaded = False`. Native is now the sole source of truth; topology is
  lazily repopulated from it on first access.

- `_replace_from`: added `self._topology_loaded = other._topology_loaded` so that
  in-place operations (flip, transform, etc.) correctly reset the topology-loaded flag
  from the replacement Brep (always False for fresh operation results). Previously,
  `_topology_loaded` was left at True from before the operation, causing subsequent
  topology access to return a stale empty list instead of repopulating from the new native.

- Removed dead `_ensure_native` method (called nowhere; embedded the old rebuild-from-
  Python-topology pattern which is now obsolete).

- Updated module and class docstrings to reflect the thin-wrapper architecture.

The `_deserialize_brep_data` helper function remains: it builds a temporary Python
topology Brep used exclusively by `brep_rebuild` to reconstruct the native OCC shape.
It will be removed in issue 04 when the serialization format changes.

All @pytest.mark.occ tests verified via CI.

All acceptance criteria for issue 03 checked off.

  shape — mirrors STEP's VERTEX_POINT / EDGE_CURVE / FACE_SURFACE entity model

  `brep_rebuild(brep, data)` directly — no silent NotImplementedError catch.

## Task: Rhino serialization parity — issue 06

Implemented `brep_to_data` and `brep_rebuild` for the Rhino backend.

New/changed code:
- `rhino_brep_to_data` in `conversion.py`: extracts vertices, edges (3D curves),
  and faces (surfaces + oriented loops + pcurves) from `Rhino.Geometry.Brep`,
  producing the same v4 STEP-inspired dict as the OCC backend.
- `_extract_trim_pcurve` in `conversion.py`: converts `Rhino.BrepTrim.TrimCurve`
  to a 2D NurbsCurve (UV parameter space, z=0), mirroring OCC's `_extract_pcurve`.
- `_rhino_curve_from_loop` in `conversion.py`: builds a joined Rhino Curve from a
  BrepLoop's trims, respecting `is_reversed` on each edge.
- `rhino_rebuild` in `operations.py`: decodes dict → Python topology → calls
  `brep_to_rhino`, symmetric to `occ_rebuild`.
- Updated `brep_to_rhino` in `conversion.py`: planar faces now use
  `_rhino_curve_from_loop` to build outer + inner boundary curves and pass all to
  `CreatePlanarBreps`, correctly reconstructing faces with holes (e.g. from boolean
  difference). Previously only outer loop vertices were used.
- `plugins.py`: replaced `NotImplementedError` stubs for `brep_to_data` and
  `brep_rebuild` with real dispatch.

New test module `tests/test_rhino_serialization.py` (14 tests, `@pytest.mark.rhino`):
- Format tests: version 4, correct entity counts for a unit box.
- Round-trip tests: unit box (volume + topology counts) and boolean-subtracted shape
  (5% relative volume tolerance).
- Cross-backend test: static OCC-produced JSON payload deserializes in Rhino.
- All 14 tests skip cleanly without rhinoinside; 114 OCC tests still pass.

All acceptance criteria for issue 06 checked off.
